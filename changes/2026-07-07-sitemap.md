# XML sitemap rewrite (2026-07-07)

## What changed

**One file: `functions.php`** — the `/sitemap.xml` generator (template_redirect
handler, section 7) was rewritten. Exact diff: [`2026-07-07-sitemap.patch`](2026-07-07-sitemap.patch).

### Contents of the new sitemap (in order)
1. **Homepage** — priority 1.0, `lastmod` = most recently modified content.
2. **All published pages** — discovered dynamically instead of the old
   hardcoded list (which pointed at `/about/` etc. whether or not those slugs
   existed). The `blog` page gets 0.9/daily; other pages 0.7/monthly.
   Password-protected pages and the front page duplicate are skipped.
   Real `lastmod` on every page.
3. **Blog posts** — 0.8/weekly with `lastmod`.

### Deliberately removed vs the old version
- **Category archives** (`/category/<slug>/`) — no category template in the
  theme (they render the bare `index.php` fallback) and the blog filters
  categories client-side without ever linking those URLs.
- **Everything annuaire-related** (per user request, same day): the
  `/annuaire/` page itself and all `/annuaire/?id=slug` agency deep links.
  The page still exists on the site; it's just not advertised to search
  engines via the sitemap.
- `specialty_hub` single permalinks — the front-end only links specialties as
  `/annuaire/?service=…` filters, so their single pages are orphans and were
  polluting the sitemap.
- The fake `changefreq daily / priority 1.0` on every static page.

### New behaviors added alongside
- `wp_sitemaps_enabled` → false: WordPress core's `/wp-sitemap.xml` is
  disabled so search engines see a single sitemap, not two competing ones.
- `robots_txt` filter appends `Sitemap: <home>/sitemap.xml` (uses `home_url()`
  so it's correct on local and on agencemarketingdigital.com without
  hardcoding). Note: this only works for WordPress's *virtual* robots.txt —
  if a physical `robots.txt` file exists on the server, add the line there
  manually instead.

## How to revert

```bash
# If not committed yet:
git checkout -- functions.php   # (only if no other functions.php edits are pending)

# Or apply the patch in reverse (safe even after other commits):
git apply -R changes/2026-07-07-sitemap.patch
```
